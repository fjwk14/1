# 汎用スキル集 (Universal Skills)

Claudeを使う誰にでも効く、ドメイン非依存の「型」5つ。
仕事の5動作 ―― **理解する / 磨く / 組む / 選ぶ / 学ぶ** ―― をカバーします。

| スキル | ディレクトリ | いつ発動するか |
|---|---|---|
| 要約 | `summarize/` | 「要約して」「まとめて」「TL;DR」 |
| 推敲 | `polish/` | 「推敲」「校正」「もっと丁寧に」 |
| 段取り | `plan/` | 「計画立てて」「何から始める」 |
| 意思決定 | `decide/` | 「どっちがいい」「決められない」 |
| かみ砕く | `explain/` | 「説明して」「かみ砕いて」 |

## 使い方

Claudeがユーザーの意図を見て**自動で呼び出す**ので、特別な操作は不要です。
明示したいときは `/summarize` のようにスラッシュでも呼べます。

## スコープ

- **このリポジトリだけで使う** → 今の場所(`.claude/skills/`)のまま。チームで共有できます。
- **全プロジェクトで使う** → 各スキルフォルダを `~/.claude/skills/` にコピー:

  ```bash
  cp -r .claude/skills/{summarize,polish,plan,decide,explain} ~/.claude/skills/
  ```

## スキルの作り方(自分で増やすとき)

`~/.claude/skills/<名前>/SKILL.md` を作り、先頭に frontmatter を書くだけ:

```markdown
---
name: my-skill
description: いつ使うかを具体的に。ここに書いた語がトリガーになる。
---

# タイトル

## 手順
1. ...
2. ...

## 原則
- ...
```

コツ: **description に「発動キーワード」を具体的に入れる**ほど、狙った場面で正しく呼ばれます。
本体は「手順」+「原則(やらないこと含む)」の2部構成が安定します。
