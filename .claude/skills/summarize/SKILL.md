# 要約 (summarize)

長い入力を、読み手が「使える」形に圧縮する。単なる短縮ではなく、**目的に合った情報の残し方**をする。

## 手順

1. **目的を1秒で見立てる** — なぜ要約が要るのか(全体把握 / 判断材料 / 共有 / 記憶の代わり)。曖昧なら短く確認、明らかならそのまま進む。
2. **階層で出す**(長さは中身に合わせて調整):
   - **一言**: 全体を1文で。
   - **要点**: 3〜7個の箇条書き。各行は「主語+結論」で完結させる。
   - **詳細**: 元が長い/複雑なときのみ。テーマ別に見出しを付けて整理。
   - **次の一手 / アクション**: 決定事項・宿題・締切があれば必ず抽出。担当と期限を落とさない。
3. **事実は忠実に** — 数字・固有名詞・日付・金額は改変しない。原文にない主張を足さない。
4. **不確かさを明示** — 情報が欠けている/矛盾している箇所は「原文からは不明」と書く。憶測で埋めない。

## 原則

- 圧縮率より「**捨てて良いものを正しく捨てる**」精度を優先する。
- 相手が次に取る行動が変わる情報(決定・数字・締切・リスク)は最後まで残す。
- チャットスレッドなら「誰が何を決めたか / 未解決の論点」を分けて示す。
