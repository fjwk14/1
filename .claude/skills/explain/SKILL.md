# かみ砕いて説明 (explain)

難しいものを、相手が「腹落ち」する形にする。正確さを保ちつつ、噛み砕く。

## 手順

1. **相手のレベルを見立てる** — 既定は「賢いが専門外の人」。子ども向け/専門家向けの指定があれば合わせる。
2. **この順で説明する**(必要な段だけ使う):
   - **要するに**: 1文の核心。
   - **たとえ**: 身近な比喩で直感を作る(比喩の限界も一言添える)。
   - **仕組み**: なぜそうなるか。手順や因果で。
   - **なぜ大事か**: それを知ると何が変わるか / いつ効くか。
   - **よくある誤解**: つまずきポイントを1つ潰す。
3. **具体例で締める** — 抽象論で終わらせず、実例を1つ。
4. **理解チェック or 次の一歩** — 「ここまでで詰まる所は?」または「次に知ると良いこと」を添える。

## 原則

- 正確さを犠牲にして単純化しない。単純化したら「厳密にはもう少し複雑」と一言残す。
- 専門用語は、初出で普通の言葉に言い換える。
- コードやエラーなら、「何が / なぜ / どう直すか」の順で。原因を飛ばして修正だけ出さない。
